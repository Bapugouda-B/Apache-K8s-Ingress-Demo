Apache Website Deployment on Kubernetes

A Will Demonstrating Kubernetes, Docker, and Ingress Setup

About the Project

This project showcases the deployment of an Apache website using Docker and Kubernetes with 2 replicas and Ingress rules for path-based routing. It's part of the transition from Monolithic to Microservices Architecture at Ventura Software.

Technologies Used

Docker

Containerizing the Apache website using a custom Dockerfile and pushing it to Docker Hub.

Kubernetes

Deploying the Dockerized Apache app on Kubernetes with multiple replicas and services.

Ingress

Setting up an Ingress Controller for routing traffic based on specific paths for multi-service architecture.

Deployment Steps

Check out the detailed deployment steps for this project:

  • Step 1: Clone the Apache website from GitHub and create a Dockerfile
  • Step 2: Build and push the Docker image to Docker Hub
  • Step 3: Create a Kubernetes deployment with 2 replicas
  • Step 4: Set up services and Ingress routing
  • Step 5: Verify the deployment and access the services
